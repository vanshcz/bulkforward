</details><details> <summary><b>⏳ Flood wait errors</b></summary>
Symptoms:

text

FloodWaitError: waiting 300 seconds
What it means:

Telegram rate limiting
Too many requests
Normal anti-spam measure
Automatic Handling:

Python

✓ Bot pauses automatically
✓ Waits required time
✓ Resumes operation
✓ Logs event
Manual Solutions:

YAML

Increase delays:
├─ Broadcast: 5-15s (from 3-8s)
├─ Replies: 3-8s (from 2-5s)
└─ Reactions: 2-5s (from 1-3s)

Reduce frequency:
├─ Cycle time: 10 min (from 4 min)
├─ Check interval: 60s (from 20s)
└─ Enable DM Only mode
Prevention:

✅ Use recommended delays
✅ Don't spam
✅ Increase cycle time
✅ Monitor statistics
